# /validate-video

技術チェック結果（`04_validation.json`）とフレーム画像を確認し、動画の見た目を視覚的に評価する。

## 手順

1. Read ツールで `.cache/pipeline/04_validation.json` を読み込み、技術チェック結果を確認する
   - `ok` が false の場合、`errors` の内容を表示して終了（視覚チェックは不要）

2. Read ツールで `04_validation.json` の `frames` フィールドに列挙されたすべての PNG ファイルを読み込む（画像として表示される）

3. 以下の観点でフレームを視覚的に評価する:
   - 字幕が読みやすく、画面内に収まっているか
   - 背景画像が真っ黒でない（Stable Diffusion 生成に成功しているか）
   - タイトルやテキストが文字化けしていないか
   - 明らかな視覚的破綻（真っ黒・真っ白・レイアウト崩れ）がないか

4. 評価結果を `04_validation.json` に追記して Write ツールで保存する:
   ```json
   {
     "visual_check": {
       "ok": true,
       "issues": [],
       "notes": "問題なし"
     }
   }
   ```
   問題がある場合は `ok` を `false`、`issues` に具体的な問題を列挙する。

5. 技術チェック・視覚チェックのいずれかが NG なら、`ok` フィールドをトップレベルでも `false` に更新して保存する。

6. 結果サマリーを出力する:
   - OK の場合: 「動画チェック OK: 技術・視覚ともに問題なし」
   - NG の場合: 「動画チェック NG: {errors と issues の一覧}」
